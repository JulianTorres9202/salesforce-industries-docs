\* If an attribute rule has Override = Yes, that means the rule was set on the instance of the child spec that's attached to the Multi Auto root product spec. That means that the instance of the child spec on Multi Auto overrides the child spec.

To learn about how child spec inheritance and overrides work, see [](https://help.salesforce.com/s?language=en_US)[Insurance Product Inheritance](https://help.salesforce.com/s/articleView?id=ind.insurance_insurance_product_inheritance_604117.htm&language=en_US&type=5 "Insurance product modeling includes two levels of inheritance: child spec inheritance and product class inheritance. By using inheritance, you can cut down the amount of time you spend individually creating every product model in your product catalog.").
