# Standard Configurator and Advanced Configurator

Product Configurator provides the Business Rules Engine and the Constraint Rules Engine. To use simple product configuration rules, use Standard Configurator. To use large and complex product configuration rules, use Advanced Configurator.

If you enable both the rules engines, you can specify which rules engine to use by default, and you can also enable sales reps to choose which rules engine to use for each transaction.

Note While Advanced Configurator and Standard Configurator can be enabled in the org at the same time, it's not recommended to change the Transaction Processing Type after quote creation. Switching between transaction types mapped to different rule engines may result in inconsistenties during rule execution.
