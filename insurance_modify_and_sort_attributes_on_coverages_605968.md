# Modify and Sort Attributes on Coverages

You can change the configuration of any attribute in a coverage on a root product spec. Your changes to a coverage on a root product override the inherited configuration from the coverage spec.

1.  On the **Coverages** tab, navigate to the coverage you want to work with.
2.  Click the name of the attribute on the coverage you want to modify.
3.  Make changes to any of the following fields (as available):
    
    -   **Value Default**
        
    -   Attribute value rules (scales icon next to available pre-configured attribute values)
        
        To learn more about attribute value rules, see [Create Attribute Value Rules](https://help.salesforce.com/s/articleView?id=ind.insurance_create_attribute_value_rules_608112.htm&language=en_US&type=5 "Use Attribute Value Rules to determine if and when an attribute value is visible to users.")
        
    -   **Value Description**
        
    -   **Read Only**
        
    -   **Rating**
        
    -   **Rating Input/Output Mapping**
        
    -   **Is Hidden**
        
    -   **Rating Attributes Use**
        
    
    The other fields on the attribute can't be modified on the root product spec.
    
    Also, you can't add values to an existing list of attribute values on the root product spec.
    
4.  After you make all the changes to one attribute, click **Save**.
5.  Repeat these steps for each attribute on each coverage that you want to modify.
6.  To sort the attributes on a coverage, drag and drop them into the order you want them to appear on the UI. Vlocity automatically saves the order you create, and honors it when users look at the coverage on a UI
