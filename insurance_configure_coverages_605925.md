# Configure Coverages

After you add coverages to a root product spec, you can configure them as child coverages, modify their attributes, make them optional, add rules to optional coverages, and re-order them in the UI.

-   Make Coverages Optional
    
    To make a coverage on a root product optional, select **Optional** on the coverage.
    
-   Re-order Coverages
    
    Drag and drop coverages in the order you want them displayed on the UI when users see the product.
    
-   [Optional Coverage Rules](https://help.salesforce.com/s/articleView?id=ind.insurance_optional_coverage_rules.htm&language=en_US&type=5 "Several types of rules apply to optional coverages on Insurance and Health products. Set rules up based on eligibility for optional coverage, whether optional coverages are required or selected by default, and valid combinations of optional coverages.")
    
    Set up rules that apply to optional coverages on Insurance products.
    

-   **[Set Up Coverages as Children of Insured Items for Multi-Instance](https://help.salesforce.com/s/articleView?id=ind.insurance_set_up_coverages_as_children_of_insured_items_for_multi_instance.htm&language=en_US&type=5)**  
    After you add coverages to a root product spec, you can associate them with an insured item.
-   **[Modify and Sort Attributes on Coverages](https://help.salesforce.com/s/articleView?id=ind.insurance_modify_and_sort_attributes_on_coverages_605968.htm&language=en_US&type=5)**  
    You can change the configuration of any attribute in a coverage on a root product spec. Your changes to a coverage on a root product override the inherited configuration from the coverage spec.
