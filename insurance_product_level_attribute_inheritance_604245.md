# Product-Level Attribute Inheritance

You can set attributes at the product level of a product class (that is, on the attribute tab on the product class spec). All product models you create based on that product class inherit these attributes.

You can add attributes to a product model that's based on a product class.

You cannot delete attributes from a product model that are inherited from a product class.

You cannot change these configuration choices on product-level attributes inherited from a product class:

-   Value Data Type
    
-   Second field of Value Data Type
    
-   Is Configurable
    
-   Attribute Group Type
    

You can make these changes to product-level attributes inherited from a product class:

-   Value
    
-   Value Description
    
-   Is Hidden
    
-   Rating
    
-   Tax and Fee Rating Attribute
    
-   Rating Attribute Use
    
-   Rating Input/Output Mapping
    
-   Tax and Fee Rating Attribute Use
    
-   Tax and Fee Rating Input/Output Mapping
    

You can revert any changes you made to product attributes. When you click revert, the attribute reverts to the configuration and rules on the product class.
