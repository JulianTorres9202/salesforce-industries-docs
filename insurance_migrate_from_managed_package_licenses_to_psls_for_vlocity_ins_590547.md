# Migrate from Managed Package Licenses to Permission Set Licenses for Insurance

You can migrate your users from Managed Package Licenses (MPLs) to permission set licenses. Also, learn how to map products or licenses (SKUs) to permission set licenses and their respective permission sets.

-   **[Set Up Sandbox Environments for Permission Set Licenses for Insurance](https://help.salesforce.com/s/articleView?id=ind.insurance_set_up_sandbox_environments_for_psls_for_vlocity_insurance_590551.htm&language=en_US&type=5)**  
    To use sandbox environments to test permission set licenses, configure your environment by creating a custom setting and running a command.
-   **[Migrate Your Licenses for Insurance](https://help.salesforce.com/s/articleView?id=ind.insurance_migrate_your_licenses_for_vlocity_insurance_590587.htm&language=en_US&type=5)**  
    Migrate your users from Managed Package Licenses (MPL) to Permission Set Licenses.
-   **[Agent Permission Sets](https://help.salesforce.com/s/articleView?id=ind.insurance_agent_permission_sets_590960.htm&language=en_US&type=5)**  
    Give your agents access to just what they need to complete their jobs with Agent Permission Sets.
-   **[Guest User Access Overview for Vlocity Package](https://help.salesforce.com/s/articleView?id=ind.insurance_guest_user_access_overview.htm&language=en_US&type=5)**  
    Guest users are unauthenticated users who can perform core actions in a Salesforce org without registering as community users.
