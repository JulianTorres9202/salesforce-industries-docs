# Cloning Products

When you've got a bunch of similar products to create, it's a lot easier to create only the first one from scratch. Then you can clone it and make the minor changes you need to the cloned product.

In Vlocity, you can clone products that include:

-   Insured item specs
    
-   Insured party specs
    
-   Coverage specs
    
-   Rating fact specs
    
-   Attributes
    
-   Rules
    
-   Details
    
-   Pricebook associations
    
-   Product child items
    
-   Taxes and fees
    

You can make changes to any of these components in your cloned product.

If you need to clone large products (dozens of coverages or hundreds of attributes or dozens of rules or all of the above), you can do that too, using Vlocity's deep clone in batch mode functionality.

-   **[Cloning Products Workflow](https://help.salesforce.com/s/articleView?id=ind.insurance_cloning_products_workflow_608604.htm&language=en_US&type=5)**  
    To make product cloning work you need to create a button and add it to the Product page layout. Then you can go to the Products page and get to work.
-   **[Create a Deep Clone Action or Button for the Products Page Layout](https://help.salesforce.com/s/articleView?id=ind.insurance_create_a_deep_clone_action_or_button_for_the_products_page_layout_608626.htm&language=en_US&type=5)**  
    You can add or create an action or button to the page layout to enable users to clone a product by clicking a button on the Product page.
-   **[Add the Deep Clone Action or Button to the Page Layout](https://help.salesforce.com/s/articleView?id=ind.insurance_add_the_deep_clone_action_or_button_to_the_page_layout_608662.htm&language=en_US&type=5)**  
    After you've created the action or button, you must add it to the Product page layout.
-   **[Add a Deep Clone in Batch Button to the Page Layout](https://help.salesforce.com/s/articleView?id=ind.insurance_add_a_deep_clone_in_batch_button_to_the_page_layout_608685.htm&language=en_US&type=5)**  
    If you have large products (dozens or hundreds of attributes, dozens or hundreds of coverages), you can add a **Deep Clone in Batch** button to your product layout. This button is configured to clone large products by initiating a batch job. This avoids timeout errors.
