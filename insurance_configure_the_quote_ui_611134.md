# Configure the Quote UI

You can configure the Quote UI to help you perform a variety of quoting functions.

The Quote UI can support basic quoting, multi-root quoting, and large group quoting if you configure it to do so. There are also several other features that can help you during the quoting process to keep it running quickly and accurately.

To get things rolling:

-   [Enable Quote UI Configuration Options](https://help.salesforce.com/s/articleView?id=ind.insurance_enable_quote_ui_configuration_options_611168.htm&language=en_US&type=5 "You can choose several Quote UI configuration options based on your business needs.")
    

Then configure your Quote UI based on your needs:

-   [Compact Quote History Timeline with Insurance History LWC](https://help.salesforce.com/s/articleView?id=ind.insurance_compact_quote_history_timeline_with_insurance_history_lwc_611225.htm&language=en_US&type=5 "You can configure the Insurance History Lightning web component (LWC) to display a timeline of quote activity in a single, chronological feed that's easy to scan and drill into. See essential activity at a glance and expand content like notes and emails to dive into more detail.")
    
-   Attribute and Attribute Value Rules Debug Mode
    
    [](https://help.salesforce.com/s?language=en_US)Getting Vlocity Insurance to run correctly can be tricky. For attribute rules and attribute value rules on quotes, you can:
    
    [](https://help.salesforce.com/s?language=en_US)
    1.  Enable rules debug mode to check your rules in real time, as they're evaluated against the attributes in any given quote. See [Debug Attribute and Attribute Value Rules](https://help.salesforce.com/s/articleView?id=ind.insurance_debug_attribute_and_attribute_value_rules_611236.htm&language=en_US&type=5 "Debug mode gives you insight into attributes and attribute rules for a quote.").
        
    2.  Troubleshoot issues that come up frequently in debug mode. See [Fix Common Issues Found in Rules Debug Mode](https://help.salesforce.com/s/articleView?id=ind.insurance_fixcommon_issues_found_in_rules_debug_mode_611350.htm&language=en_US&type=5 "There are a few issues that often come up when you're running rules debug mode. Check for the usual suspects at the root of the problem and take steps to fix things.").
        
-   [Root Product Cloning](https://help.salesforce.com/s/articleView?id=ind.insurance_root_product_cloning_611399.htm&language=en_US&type=5 "You can clone root products on the Quote UI if the Enable Product Cloning option is selected.")
    
-   [Edit Panel Mode](https://help.salesforce.com/s/articleView?id=ind.insurance_edit_panel_mode_611413.htm&language=en_US&type=5 "The Edit Panel Mode is designed for specific use cases where products are highly configurable and can have a large number of coverages and attributes.")
    
-   [Group Classes for Large Group Quotes](https://help.salesforce.com/s/articleView?id=ind.insurance_group_classes_for_large_group_quotes_611436.htm&language=en_US&type=5 "Group classes describe a group of plan subscribers who'll all receive similar or the same benefits. You can add them to plans on large group quotes.")

## What's Next

If you want to use your own styles and/or add functionality to these processes, you can create an OmniScript to customize the quoting experience for your users.

To learn more about using OmniScripts on the Quote UI, see:

-   [Use OmniScripts to Add and Modify Insured Items](https://help.salesforce.com/s/articleView?id=ind.insurance_use_omniscripts_to_add_and_modify_insured_items_614083.htm&language=en_US&type=5 "Vlocity Insurance offers an out-of-the-box form your users can use to add and modify insured items on the Quote UI. However, this form isn't customizable. If you want to use your own styles or add functionality to these processes, you can create an OmniScript and hook up that OmniScript to the Quote UI.")
