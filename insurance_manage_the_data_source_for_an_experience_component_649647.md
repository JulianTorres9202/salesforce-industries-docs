# Manage the Data Source for an Experience Component

Some Experience components use data sources to pull in information to display.

Note If your experience component doesn't have or doesn't need a data source, you can skip this topic.

The highlighted are in this flow shows where you are in the process of customizing Experience components:

Note

Most data sources used by experience components are one of these:

-   Integration Procedure
    
-   Omnistudio Data Mapper
    
-   Apex Remote used for a Vlocity Insurance service
    

Before You Begin

-   Determine what data source you plan to use.
    
-   If you're using a Vlocity Insurance service or one of the out-of-the-box Data Mappers or Integration Procedures, determine what remote options and other settings you want to use, and what values to assign to each.
    
-   If you're using a totally custom Data Mapper, Integration Procedure, or other data source, make sure it's created and tested. Determine what options and settings you need, and what values to assign to each.
