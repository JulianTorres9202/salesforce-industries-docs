# Post-Install Steps for Vlocity Insurance Application Suite Group Voluntary Benefits

Before exploring your shiny, new Group Voluntary Benefits Application org, you need to complete a few configuration tasks to make sure you don't run into any glitches while you're doing your thing. This section walks you through all the post-deployment steps that you must complete.

You've likely got an email with the org URL and login credentials. Use that information to log in to your org and:

-   Configure Your Digital Experience Sites After Vlocity Insurance Application Suite Group Voluntary Benefits Installation
    
-   Set up the Google Maps API Key (optional)
    
-   [Configure Sharing Rules for Site Users](https://help.salesforce.com/s/articleView?id=ind.insurance_configure_sharing_rules_for_site_users_omnistudio.htm&language=en_US&type=5 "Sharing rules give your site users greater access by making automatic exceptions to your org-wide sharing settings.")
    

-   **[Configure Your Digital Experience Sites After Vlocity Insurance Application Suite Group Voluntary Benefits Installation](https://help.salesforce.com/s/articleView?id=ind.insurance_configure_digital_experience_sites_after_vlocityins_app_suite_gvb_install_omnistudio.htm&language=en_US&type=5)**  
    There are a few configuration steps for the Digital Experience Sites that you create for your users.
-   **[Set up the Google Maps API Key](https://help.salesforce.com/s/articleView?id=ind.insurance_set_up_the_google_maps_api_key_omnistudio.htm&language=en_US&type=5)**  
    Did you know that the Group Voluntary Benefits Application supports Google Map APIs to automatically populate address fields?
-   **[Configure Sharing Rules for Site Users](https://help.salesforce.com/s/articleView?id=ind.insurance_configure_sharing_rules_for_site_users_omnistudio.htm&language=en_US&type=5)**  
    Sharing rules give your site users greater access by making automatic exceptions to your org-wide sharing settings.
