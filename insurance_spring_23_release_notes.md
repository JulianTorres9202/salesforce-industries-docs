# Insurance Spring '23 Release Notes

Expedite financial authorization requests with a workflow that routes them directly to supervisors. Define user-specific attribute values with profile-based attribute rules. Backdate policy changes with out-of-sequence endorsement quoting. Delete census data for group benefits directly on the quote record page with the delete functions on the Insurance Census Lightning web component.

-   **[Automatic Upgrades](https://help.salesforce.com/s/articleView?id=ind.insurance_automatic_upgrades.htm&language=en_US&type=5)**  
    Starting in Spring '23, we're moving our Insurance managed package to the Salesforce automatic upgrade process. This process aligns with the Salesforce seasonal release cycle, where new features are delivered three times a year (Summer, Winter, and Spring). With automatic upgrades, you also get early access to upcoming releases in your sandbox preview environment.
-   **[Required Upgrade to OAuth 2.0 for DocuSign Integration in Insurance](https://help.salesforce.com/s/articleView?id=ind.insurance_required_upgrade_to_oauth_20_for_docusign.htm&language=en_US&type=5#insurance_required_upgrade_to_oauth_20_for_docusign)**  
    For Vlocity Document Generation, you must upgrade to OAuth 2.0 to manage DocuSign.
-   **[Build Business Processes for Financial Transaction Authorizations](https://help.salesforce.com/s/articleView?id=ind.insurance_automate_payment_authorizations_for_claims.htm&language=en_US&type=5)**  
    Set limits on how much users can authorize for claim payments and build a workflow that routes payment authorization requests to supervisors. Users authorized to approve larger amounts receive notifications of pending requests, link directly to claim details, and approve or reject requests to complete financial transactions.
-   **[Apply Rules to Deeper Levels of the Product Hierarchy](https://help.salesforce.com/s/articleView?id=ind.insurance_apply_rules_to_deeper_levels_of_the_product_hierarchy.htm&language=en_US&type=5)**  
    We've improved the way insurance rules are evaluated for more complex insurance products. OmniScript Lightning web components support optional coverage and attribute rules evaluation for coverages in the lowest-level structures of the product hierarchy. Rules are applied consistently to coverages in the lowest-level structures and across multiple insured items in the same level of the product hierarchy. These changes make rules run more efficiently in Lightning web components.
-   **[Customize Attribute Values Based On User Profiles](https://help.salesforce.com/s/articleView?id=ind.insurance_designate_attribute_availability_with_profile_based_attribute_rules_.htm&language=en_US&type=5)**  
    Show users unique product attribute values based on their user profile. For example, when a Customer Service Representative creates a travel insurance quote, set the luggage loss coverage limit to $250 by using attribute rules. When a Captive Agent creates a quote, set the same coverage limit to $500.
-   **[Delete Census Members for a Group Benefits Quote](https://help.salesforce.com/s/articleView?id=ind.insurance_delete_census_members_for_a_group_benefits_quote.htm&language=en_US&type=5)**  
    Use the Insurance Quote Census Lightning web component in the Salesforce data model to delete members on a Quote record page. You can delete a single census member record or multiple member records using the Delete Members option. The hierarchical tree view helps brokers and insurance carrier administrators browse through large, detailed data sets more efficiently.
-   **[Issue an Out-of-Sequence Endorsement](https://help.salesforce.com/s/articleView?id=ind.insurance_issue_an_out_of_sequence_endorsement.htm&language=en_US&type=5)**  
    Create endorsements for transactions that have an effective date prior to the effective date of another active transaction on the same policy. All the backdated changes are automatically applied on top of existing policy endorsements within the term of the policy.
-   **[Open Coverages Easily for More Policy Types](https://help.salesforce.com/s/articleView?id=ind.insurance_open_coverages_easily_for_more_policy_types.htm&language=en_US&type=5)**  
    Enhancements simplify the open coverage process. A narrower list of claimants includes only the claim participants associated with claim items. If a coverage's insured item is a policy participant, it's saved to the claim coverage record the same way a policy asset is saved to the claim coverage record. And you can now open coverages for products with coverages only at the root level.
